# Бібліотека рецептів

Консольна утиліта для зберігання, пошуку та керування кулінарними рецептами.
Виконано в рамках навчальної практики (Варіант 11), спеціальність 121/F2
«Інженерія програмного забезпечення».

## Опис задачі

Утиліта дозволяє користувачу зберігати рецепти (назва, інгредієнти, час
приготування) та працювати з ними між сесіями запуску програми. Усі дані
зберігаються у файлі `recipes.json` у тій же директорії, що й програма.

## Функції

| № | Функція | Опис |
|---|---------|------|
| 1 | Додавання рецепту | Введення назви, переліку інгредієнтів та часу приготування |
| 2 | Пошук за назвою | Частковий пошук без урахування регістру |
| 3 | Пошук за інгредієнтом | Частковий пошук серед інгредієнтів усіх рецептів |
| 4 | Перегляд усіх рецептів | Виводить повний список збережених рецептів |
| 5 | Видалення рецепту | Видалення за номером зі списку |

## Формат даних

Дані зберігаються у JSON-форматі, масив об'єктів:

```json
[
  {
    "name": "Борщ",
    "ingredients": ["буряк", "капуста", "морква", "м'ясо"],
    "time_minutes": 90
  }
]
```

## Структура проєкту

```
recipe-book/
├── main.py                   # Точка входу, консольне меню, обробка вводу
├── recipe_storage.py         # Логіка зберігання та обробки рецептів
├── test_recipe_storage.py    # Автоматизований тест (бонус)
├── recipes.json              # Файл даних (створюється автоматично при першому запуску)
├── .gitignore
└── README.md
```

## Як запустити

Потрібен Python 3.8 або новіший. Зовнішні бібліотеки не використовуються.

```bash
python3 main.py
```

## Як запустити автотест

```bash
python3 test_recipe_storage.py
```

Очікуваний результат: `УСІ ТЕСТИ ПРОЙШЛИ УСПІШНО`.

## Обробка некоректного вводу

- Порожні поля при додаванні рецепту запитуються повторно.
- Нечислове або від'ємне значення часу приготування не приймається —
  програма просить ввести коректне число знову.
- Невірний пункт меню повідомляє про помилку та повертає до меню,
  без аварійного завершення.
- Пошкоджений або відсутній файл `recipes.json` не призводить до збою —
  програма починає роботу з порожньою бібліотекою.
- Вихід через `Ctrl+C` обробляється коректно (без traceback).

## Приклад сценарію тестування (для звіту, розділ 5)

| № | Сценарій | Дія | Очікуваний результат | Фактичний результат |
|---|----------|-----|----------------------|----------------------|
| 1 | Додавання рецепту | Меню → 1 → "Борщ" → інгредієнти → "90" | Рецепт додано і збережено у файл | ОК |
| 2 | Пошук за назвою | Меню → 2 → "борщ" | Знайдено 1 рецепт | ОК |
| 3 | Пошук за інгредієнтом | Меню → 3 → "буряк" | Знайдено 1 рецепт | ОК |
| 4 | Перегляд усіх | Меню → 4 | Виведено список усіх рецептів | ОК |
| 5 | Видалення рецепту | Меню → 5 → номер рецепту | Рецепт видалено зі списку та з файлу | ОК |
| 6 | Некоректний час приготування | Введення "абв" замість числа | Запит повторюється, програма не падає | ОК |
| 7 | Від'ємний час приготування | Введення "-5" | Запит повторюється з поясненням | ОК |
| 8 | Невірний пункт меню | Введення "9" у головному меню | Повідомлення про помилку, повернення в меню | ОК |
| 9 | Видалення неіснуючого індексу | Введення номера поза межами списку | Повідомлення "не знайдено", без збою | ОК |
| 10 | Перезапуск програми | Закрити та відкрити програму знову | Раніше додані рецепти збережені | ОК |
